CI/CD pipeline workflows

An example CI/CD pipeline for deploy multiple branches on GitHub Pages using GitHub Actions workflows. This involves using a single repository with multiple branches (e.g., main, staging, dev etc.) to trigger different actions and leveraging a specific action to deploy to GitHub Pages.

URLs

            https://{github-user}.github.io/{repository}/
            https://{github-user}.github.io/{repository}/{branch-name}/
            https://{github-user}.github.io/{repository}/{another-branch-name}/

For example, I have created main, staging, gh-pages branches.

  • main - for prodution with url:
    https://pine1611.github.io/deploy-multi-branches/
  • staging - for testing deploy before merge to main with url:
    https://pine1611.github.io/deploy-multi-branches/staging/
  • gh-pages - environment for deploy
